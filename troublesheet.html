⊕ ¿Qué revisar en cada capa TCP/IP?

Aplicación: ¿El servicio está corriendo? ¿Puerto abierto? ¿Cert TLS válido?

Transporte: ¿TCP/UDP al puerto correcto? ¿Firewall intermedio? ¿SYN/ACK fluyen?

Internet: ¿Hay ruta? ¿Ping al destino? ¿Routing asimétrico?

Acceso a red: ¿Link up? ¿ARP resuelve? ¿VLAN/tag correcto?

⊟ Protocolos clave — qué son y cuándo sospechar

ARP (Address Resolution Protocol)

Resuelve IP → MAC. Si falla, tenés un problema L2 o de VLAN. Comando clave: arp -a, show arp.

ICMP (Internet Control Message Protocol)

Lo que usa ping y traceroute. Si ping falla pero traceroute muestra saltos, hay un firewall bloqueando ICMP, no routing roto.

TCP vs UDP

TCP = confiable, handshake 3-way. UDP = sin conexión, rápido.

DNS

Resuelve nombres → IPs. Falla típica: ping al IP anda, ping al hostname no. Revisar /etc/resolv.conf, nslookup, dig.

⎘ Three-way handshake — TCP

Antes de mandar datos, TCP negocia:

  1. SYN — el cliente pide conexión
  2. SYN-ACK — el servidor acepta y pide su turno
  3. ACK — el cliente confirma
Si ves SYN retransmitido sin SYN-ACK: el server no escucha, firewall, RST, o MTU blackhole. 
tcpdump -i eth0 -nn -S port 80
# Flags: [S] = SYN, [S.] = SYN-ACK, [.] = ACK, [R] = RST, [F] = FIN

▣ ¿Qué es una VLAN?

Una VLAN = un broadcast domain independiente.

Beneficio 1: Reducir tamaño del broadcast domain.
Beneficio 2: Seguridad: aislar grupos.
Beneficio 3: Flexibilidad al mover usuarios.

▣ ¿Por qué telecom en una guía de redes?

La fibra y el cobre no llegan a todos lados: zonas rurales, mar, montañas, plantas industriales distribuidas. Una red moderna combina cableado + inalámbrico.

Cuándo usar radio: distancias de 100m a 50+ km sin fibra, topologías rurales, redundancia de uplink.
Cuándo NO usar radio: alta densidad, interferencia RF masiva, requisitos de QoS estrictos.

▸ Conceptos clave compartidos

  • Línea de vista (LOS): las antenas se "ven" sin obstáculos. Requisito #1 de cualquier radioenlace.
  • Zona de Fresnel: elipsoide entre antenas. Si hay obstáculos dentro del 60%, el enlace se degrada.
  • PIRE / EIRP: Potencia Isótropa Radiada Equivalente. Suma ganancia de antena + pérdidas de cable.
  • RSSI / SNR: intensidad de señal y relación señal/ruido.
  • Modulación adaptativa: baja automáticamente de 256-QAM a QPSK cuando la señal empeora.
⊕ Radioenlace Punto a Punto (PTP)

Un enlace dedicado entre dos puntos con antenas direccionales. Es la forma más común de conectar dos edificios o dos torres.

Bandas de frecuencia

BandaCaracterística
5 GHzLicencia exenta, alcance medio, sensible a lluvia
23 GHzMicroondas, requiere licencia, alta capacidad, corto alcance
60 GHzV-band, multigigabit, alcance muy corto
70/80 GHzE-band, 1–10 Gbps, distancia corta
6-42 GHz licenciadasCarriers, sin interferencia

Equipos típicos

  • Mikrotik: Wireless Wire, NetMetal — para WISP y enlaces económicos
  • Ubiquiti: airFiber, PowerBeam — buen precio/rendimiento
  • Cambium Networks: ePMP, PTP 650/670 — referencia para WISP profesional
  • Cisco / Ericsson / Nokia: soluciones carrier

Componentes de un enlace

  • ODU (Outdoor Unit): la radio propiamente dicha
  • Antena: direccional, parábola o planar. Ganancia en dBi
  • Cable guíaondas / coaxil / Ethernet: conexión ODU-IDU
  • IDU (Indoor Unit): conversor a Ethernet/FO, PoE, gestión
  • Torres / herrajes: soporte físico

Troubleshooting típico

  • Throughput bajo: revisar alineación, RSSI/SNR, modulación actual
  • Caídas cada X tiempo: interferencia horaria, saturación, batería far end
  • Link no levanta: antena desalineada, problema eléctrico, config errónea
  • Pérdida con lluvia: rain fade, especialmente en 23+ GHz

⊟ PMP — Punto a Multipunto

Un Access Point (AP) central con antena sectorial que sirve a múltiples suscriptores (CPE/SM). Modelo clásico de un WISP.

Componentes

  • AP / Base Station: en torre, antena sectorial (60°, 90°, 120°)
  • SM / CPE: antena del cliente, direccional apuntando al AP
  • GPS Sync: sincroniza APs en misma torre para que no se interfieran
  • Backhaul: conexión AP-core (fibra, otro PTP, satélite)

Planificación

  • Sectorización: 3-4 sectores por torre, frecuencias distintas
  • Capacidad: throughput agregado se reparte entre usuarios
  • Throughput real depende de modulación, distancia, interferencia

Tecnologías de acceso

EstándarBandaUso
802.11 Wi-Fi2.4/5/6 GHzLocal, low-cost
802.16 WiMAX2-66 GHzHistórico, en desuso
LTE / 5G NR700 MHz-6 GHzCarrier, eNodeB + UE
Proprietary (Cambium, Mimosa)5/60 GHzWISP, TDMA

Troubleshooting

  • SM no se asocia: revisar controlador, autenticación, distancia
  • Throughput bajo global: SNR de los SMs, interferencia entre sectores
  • Caídas masivas: problema en AP, GPS sync, alimentación
⎘ VHF / UHF / HF — Radio convencional

Las bandas "clásicas" de radio: HF (3-30 MHz), VHF (30-300 MHz), UHF (300 MHz-3 GHz). Marina, aviación, emergencias, radioafición, militar.

Bandas y propagación

BandaPropagaciónUso
HFIonosférica (rebota en ionosfera)Radioafición, broadcasting internacional
VHFLínea de vistaMarina, aviación, FM comercial
UHFLOS, mejor en ciudadesTV digital, trunking, GPS

Sistemas digitales modernos

  • DMR: dos time slots TDMA en 12.5 kHz
  • P25: emergencias y seguridad pública
  • TETRA: trunking digital europeo
  • dPMR / NXDN: FDMA digital económico

Conceptos importantes

  • Repetidora: receptor+transmisor en torre, repite señal
  • Trunking: asigna canales dinámicamente
  • CTCSS / DCS: sub-tonos para que varios grupos compartan frecuencia
  • Canalización: 12.5 kHz (narrowband) o 25 kHz (wideband)
  • Cobertura: modelada con software (Radio Mobile, ICS Telecom)

Troubleshooting

  • No se escucha la repetidora: frecuencia, offset, sub-tono, antena
  • Audio cortado/robotizado: RSSI bajo, o clipping por señal muy fuerte
  • Interferencia: otro servicio en la banda, ruido industrial, splatter adyacente
🛰 Satelital — VSAT, GEO y LEO

Para lugares donde no llega nada: mar, desiertos, zonas selváticas, aeronaves, disaster recovery. Cobertura global a cambio de latencia y costo.

Tipos de órbita

TipoAltitudLatenciaEjemplos
LEO300–2000 km20–50 msStarlink, OneWeb, Kuiper
MEO5000–20000 km100–150 msO3b, GPS, Galileo
GEO35.786 km600+ msHughes, iDirect, VSAT

VSAT

  • Antena parabólica chica (0.6 a 2.4 m)
  • Modem satelital (BUC + LNB, o modem integrado)
  • Hub central en teleport del operador
  • Bandas: Ku (12-18 GHz), Ka (26-40 GHz), C (4-8 GHz)

Cuándo usar cada uno

  • GEO/VSAT: empresarial estable, voz, datos críticos
  • LEO (Starlink): residencial, internet genérico, baja latencia
  • MEO (O3b): marítimo, cruceros, offshore
  • Iridium / Inmarsat: global extremo, handy satelitales

Troubleshooting

  • No link: antena desalineada, BUC sin alimentación, vista obstruida
  • Lento con lluvia: rain fade, especialmente en Ka
  • Ping alto en GEO: es normal (600+ ms), es física
  • Caídas intermitentes: satélite cambiando de beam (LEO), handover entre haces
